【入社課題】
一覧、詳細、作成、編集、削除の基本機能を備えています。トップページではなく、投稿一覧ページの /posts から利用します。

## 使い方

### 前提

- Docker が利用できること
- リポジトリをクローン済みであること
- WSL を使う場合は Docker Desktop の WSL Integration を有効にし、`docker` グループに所属していること(Windowsの場合はWSLを使わないと動作がかなり遅かった。)

### 初回セットアップ

```bash
git clone https://github.com/KaiseiKono/my-laravel-blog
cd my-laravel-blog
cp .env.example .env
# ポートが競合する場合は、コピーした.envの該当箇所を空いているポート番号に書き換える。


docker compose up -d
cp src/my-laravel-blog/.env.example src/my-laravel-blog/.env

docker compose exec php bash
cd my-laravel-blog
composer install
php artisan key:generate
php artisan migrate
exit

docker compose exec node sh
cd my-laravel-blog
npm install
npm run build
exit
```

### アクセス先

- 投稿一覧: http://localhost/posts
- 投稿作成: http://localhost/posts/create
- 投稿詳細: http://localhost/posts/{id}
- 投稿編集: http://localhost/posts/{id}/edit

### テスト

テストは PHP コンテナの中で実行します。

- `tests/Feature/Auth/` : Breeze標準の登録・ログイン・ログアウト等
- `tests/Feature/PostCrudTest.php` : 投稿のCRUD、ゲスト/ログインユーザー/投稿者本人/他ユーザーそれぞれの権限テスト

### 実行方法

```bash
docker compose exec php bash
cd my-laravel-blog
php artisan test
```

## 機能一覧

- 投稿を作成できる
- 投稿を一覧表示できる
- 投稿の詳細を確認できる
- 投稿のタイトルと内容を編集できる
- 投稿を削除できる

## URL設計

| URL              | メソッド | 処理                     |
| ---------------- | -------- | ------------------------ |
| /posts           | GET      | 投稿一覧ページを表示する |
| /posts/create    | GET      | 投稿作成ページを表示する |
| /posts           | POST     | 投稿作成処理を実行する   |
| /posts/{id}      | GET      | 投稿詳細ページを表示する |
| /posts/{id}/edit | GET      | 投稿編集ページを表示する |

## バリデーション

- タイトルは必須入力
- 内容は必須入力

## ユーザー登録・ログイン・ログアウトの操作方法

- ユーザー登録にはユーザー名・メールアドレス・パスワードの設定が必要。
- ログインには登録時に設定したメールアドレスとパスワードの入力が必要。
- ログイン状態でヘッダーのログアウトボタンを押すことでログアウトができる。

## 認証・認可

- ユーザー登録・ログイン・ログアウトは Laravel Breeze を使用
- 投稿の閲覧(一覧・詳細)は未ログインでも可能
- 投稿の作成・編集・削除にはログインが必要
- 投稿の編集・更新・削除は投稿者本人のみ可能（他人の投稿を操作しようとすると403）

## トラブルシューティング

### phpMyAdminにアクセスすると "Error during session start" と表示される

`docker-compose up` 初回起動時、`phpmyadmin/sessions` ディレクトリの
所有者が `root` になり、コンテナ内から書き込めずエラーになることがあります。

以下を実行して権限を修正してください。

```bash
sudo chmod -R 777 phpmyadmin/sessions
docker-compose restart phpmyadmin
```
