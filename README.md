【入社課題】
一覧、詳細、作成、編集、削除の基本機能を備えています。トップページではなく、投稿一覧ページの /posts から利用します。

## 使い方

### 前提

- Docker が利用できること
- リポジトリをクローン済みであること
- WSL を使う場合は Docker Desktop の WSL Integration を有効にし、`docker` グループに所属していること(Windowsの場合はWSLを使わないと動作がかなり遅かった。)

### 初回セットアップ

リポジトリのルートディレクトリで、次の順番で実行します。

```bash
docker compose up -d
cp src/my-laravel-blog/.env.example src/my-laravel-blog/.env

docker compose exec php bash
cd my-laravel-blog
composer install
php artisan key:generate
php artisan migrate
exit

docker compose exec node sh
cd my-laravel-blog
npm install
npm run build
exit
```

### アクセス先

- 投稿一覧: http://localhost/posts
- 投稿作成: http://localhost/posts/create
- 投稿詳細: http://localhost/posts/{id}
- 投稿編集: http://localhost/posts/{id}/edit

### テストの実行方法

テストは PHP コンテナの中で実行します。

```bash
docker compose exec php bash
cd my-laravel-blog
php artisan test --filter=PostCrudTest
```

## 機能一覧

- 投稿を作成できる
- 投稿を一覧表示できる
- 投稿の詳細を確認できる
- 投稿のタイトルと内容を編集できる
- 投稿を削除できる

## URL設計

| URL              | メソッド | 処理                     |
| ---------------- | -------- | ------------------------ |
| /posts           | GET      | 投稿一覧ページを表示する |
| /posts/create    | GET      | 投稿作成ページを表示する |
| /posts           | POST     | 投稿作成処理を実行する   |
| /posts/{id}      | GET      | 投稿詳細ページを表示する |
| /posts/{id}/edit | GET      | 投稿編集ページを表示する |
| /posts/{id}      | PUT      | 投稿編集処理を実行する   |
| /posts/{id}      | DELETE   | 投稿削除処理を実行する   |

## データベース設計

### 投稿（posts）テーブル

| カラム論理名 | カラム物理名 | 型           | 意味             |
| ------------ | ------------ | ------------ | ---------------- |
| ID           | id           | BIGINT       | 自動採番の主キー |
| タイトル     | title        | VARCHAR(255) | 投稿タイトル     |
| 内容         | content      | TEXT         | 投稿本文         |
| 作成日       | created_at   | TIMESTAMP    | 作成日時         |
| 更新日       | updated_at   | TIMESTAMP    | 更新日時         |
